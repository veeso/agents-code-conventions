# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This repository provides two Claude Code skills that enforce Rust coding conventions:

- **`rust-conventions`** — Enforces Microsoft Rust guidelines (`M-` prefix) and custom extensions (`X-` prefix) when writing or modifying `.rs` files.
- **`fetch-rust-conventions`** — Syncs `M-` guidelines from the upstream [microsoft/rust-guidelines](https://github.com/microsoft/rust-guidelines) repository.

This is a skill distribution repository, not a Rust application. There is no Cargo.toml, no build system, and no test suite.

## Key Commands

```bash
# Update Microsoft guidelines from upstream
bash scripts/fetch-guidelines.sh
```

## Architecture

### Two-Skill System

| Directory            | Skill Name               | Purpose                                                             |
| -------------------- | ------------------------ | ------------------------------------------------------------------- |
| `rust-conventions/`  | `rust-conventions`       | Reads guideline files and enforces them during Rust code operations |
| `fetch-conventions/` | `fetch-rust-conventions` | Runs the fetch script to update upstream guidelines                 |

### Guideline Prefix System

- **`M-` prefix**: Microsoft upstream guidelines in `rust-conventions/rust-guidelines.txt`. Auto-generated by `scripts/fetch-guidelines.sh` — never edit manually.
- **`X-` prefix**: Custom extensions in `rust-conventions/custom-guidelines.txt`. Maintained independently; never overwritten by the fetch script.

### File Roles

- `rust-conventions/rust-guidelines.txt` — Large generated file (~2500 lines) concatenated from Microsoft's mdBook source with `{{#include}}` directives resolved. Do not hand-edit.
- `rust-conventions/custom-guidelines.txt` — User-maintained custom conventions. Add new `X-` conventions here following the existing format.
- `scripts/fetch-guidelines.sh` — Clones the Microsoft repo into a temp directory, resolves mdBook includes, and outputs the flat guidelines file. Handles macOS/Linux differences for `sed`.

### Adding Custom Conventions

New `X-` conventions go in `rust-conventions/custom-guidelines.txt` using this format:

```markdown
## Convention Name (X-MY-CONVENTION) { #X-MY-CONVENTION }

<why>Reason this convention exists.</why>
<version>1.0</version>

Description of the convention.
```
